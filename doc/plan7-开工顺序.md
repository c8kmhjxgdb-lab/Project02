# 阶段7开工顺序（现状版）

本文档用于把 `doc/plan7.md` 拆成当前可继续执行的开发批次。  
截至 2026-06-08，阶段7已经不再是从零开工状态：童心、基地、建造、库存、玩具、任务和存档都已有可运行骨架，后续重点应从“新增模块”转为“补配置读取、拆分主循环、完善 UI/任务/小游戏闭环”。

## 当前总体状态

已落地：

- `EmotionSystem` 已新增 `childlikeHeart`，保留 `grievance/joy/stress`，低童心和高委屈都会影响后处理与速度。
- `TimeSystem` 已支持时间段、整点/日切标记、`restUntil()`。
- `WeatherSystem` 已支持区域上下文、室内天气粒子过滤、天气名称和字符串解析。
- `RegionManager` 已有 `home_base`，玩家可从新手村 `player_home` 进入并从基地 `base_exit` 返回。
- `BuildingSystem` / `Furniture` 已存在，支持 7 种家具、网格预览、旋转、放置、拆除、独立家具占用层、阻挡家具 Box2D 静态体。
- `assets/scripts/furniture.lua` 已作为家具定义来源读取。
- `Inventory` 已存在，运行时金币以库存系统为准，支持家具库存、家具解锁、购买、放置消耗、拆除返还。
- `SaveData` 已升级到 `version = 3`，保存童心、委屈、快乐、压力、时间、天气、基地家具、家具库存、家具解锁和玩具数据。
- `ToySystem` 已存在，加载 `assets/scripts/toys.lua`，模型车小游戏可进入、完成、退出，并有每日奖励限制、最佳时间、金币/童心/好感奖励。
- `QuestSystem` 已存在，加载 `assets/scripts/quests.lua`，当前 5 个基地任务可按快照条件自动完成并发奖励。
- HUD 已显示童心、委屈、时间、天气、区域、基地舒适度/童年收藏分/灯光分、建造库存/价格/锁定状态、玩具状态和任务数量。
- 新增配置资产：`assets/scripts/childhood_config.lua`、`assets/scripts/time_events.lua`、`assets/scripts/weather_config.lua`。
- Release / Debug 构建已通过。

仍需注意：

- `main.cpp` 仍承担较多阶段7调度逻辑，后续不要继续把规则堆进去。
- `childhood_config.lua`、`time_events.lua`、`weather_config.lua` 目前已落地为资产，但尚未被对应 C++ 系统读取。
- `MiniCarGame.*`、`BaseRating.*`、`EnvironmentRules.*` 还没有独立拆分。
- 商店仍是建造模式中的快捷购买，不是完整店铺 UI。
- 任务是轻量快照驱动，尚未实现完整 `Hidden/Available/Active/Completed/Rewarded` 状态机。
- 没有自动测试套件，仍以 Release/Debug 构建和手动游玩验证为准。

## 当前执行原则

1. 不重复新建已存在模块；先补现有系统的配置读取、状态拆分和交互闭环。
2. 后续新增 `.cpp` 必须加入 `CMakeLists.txt`。
3. 不修改 `build/` 生成物作为源码内容；新增脚本后用 `cmake -B build` 触发复制。
4. 优先把规则从 `main.cpp` 抽到系统模块，特别是基地评分、小游戏、环境规则和任务事件。
5. 每次改动后至少跑 `cmake --build build --config Release`；涉及 Debug 快捷键时同时跑 Debug 构建。

## 当前验收结论

阶段7 P0/P1 的核心闭环已经具备雏形：

玩家可以进入秘密基地，使用金币/库存购买并放置童年家具，家具会影响基地评分和恢复效果；童心、时间、天气、家具、库存、玩具会进入存档；模型车小游戏和 5 个基地任务已经能产生金币/童心/好感等奖励。

阶段7尚未完成的重点不是“有没有系统”，而是“系统是否足够独立、配置化、可展示、可验证”。后续开发应优先减少 `main.cpp` 压力并补齐 UI、配置读取和手动验收记录。
