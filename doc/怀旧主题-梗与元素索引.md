# 《星愿之子》怀旧主题 — 记忆意象与玩法机制索引

> 重写目标：本文件不再作为“经典作品名录”使用，而是作为“怀旧记忆如何转化为原创玩法”的设计库。  
> 使用原则：不直接使用现实游戏、动画、品牌、角色的正版称呼；只保留体验、情绪、视觉结构和机制含义。

---

## 1. 使用规则

每个怀旧元素进入游戏前，都必须经过四步转换：

```text
真实记忆
→ 情绪含义
→ 玩法机制
→ 原创命名与资产提示词
```

错误做法：

```text
把某个真实作品名写进地图、NPC、技能或 BOSS 名称。
```

正确做法：

```text
提炼它带来的玩家体验：
等待、黑屏、弹窗、课间、改写结局、断更、试错、跑调、被催促。
然后让这个体验变成关卡规则。
```

## 2. 命名规范

正式游戏文本、图片提示词、视频提示词中禁止直接出现：

- 现实平台名、网站名、厂牌名。
- 现实动画、漫画、电影、游戏名称。
- 现实角色名、英雄名、吉祥物名。
- 真实商标、logo、包装、台标。
- 可读的现实广告语。

建议使用抽象命名：

| 记忆类型 | 不建议方向 | 建议原创命名方向 |
|----------|------------|------------------|
| 网页小游戏 | 直接写真实网站名 | 弹窗游乐厅、试玩大厅、像素柜台 |
| 氪金诱导 | 直接写真实广告语 | 六元冠冕、变强按钮、闪光礼包 |
| 电视黑屏 | 直接写频道或节目名 | 雪花放映厅、静屏馆长、缺帧胶卷 |
| 超级英雄 | 直接写英雄名字 | 纸皮英雄、星盾队、披风试炼 |
| 童话故事 | 直接写经典角色名 | 反面童话、命运书签、删改皇后 |
| 学校压力 | 直接写现实机构名 | 铃声操场、试卷墙、补课总监 |
| 童谣音乐 | 直接写歌词 | 跑调童谣镇、节拍铃、噪声公爵 |
| 漫画追更 | 直接写作品名 | 断格漫画城、未完页、断更书记 |
| 科学幻想 | 直接写名人名 | 问号实验室、错误桥、标准答案博士 |
| 长大压力 | 直接写现实公司 | 灰钟局、格式化成长、催熟电梯 |

## 3. 记忆机制库

### 3.1 弹窗与试玩记忆

| 记忆意象 | 情绪含义 | 玩法机制 | 原创命名 |
|----------|----------|----------|----------|
| 免费小游戏入口 | 放学后立刻开始玩的自由 | 小型挑战房，完成后恢复童心 | 试玩门 |
| 加载条卡住 | 等待、期待、焦急 | 进度条机关，需要防守直到加载完成 | 慢速加载井 |
| 弹窗遮挡 | 快乐被打断 | 半透明窗口遮挡视野和碰撞路线 | 弹窗怪 |
| 变强按钮 | 短期诱惑 | 点击获得短暂攻击力，扣童心值 | 闪光按钮 |
| 过期插件 | 童年入口消失 | 地图块闪烁、需要修复核心才能通过 | 熄灯播放器 |
| 试玩结束 | 被迫停止 | 倒计时后关闭技能，需用试玩币续时 | 试玩倒计时 |
| 旧键盘操作 | 熟悉手感 | WASD 机关、方向键节奏门 | 键帽迷宫 |

设计要点：

- 不要让弹窗只做视觉遮挡，它应该改变战斗节奏。
- 变强诱导必须是选择题：玩家可以点，但要承担童心损失。
- 试玩币不应只是货币，也可以是打开“再玩一次”的机会。

```text
English: original top-down 2D action RPG level, colorful pop-up arcade hall, floating pop-up monsters blocking paths, glowing trial doors and loading-bar mechanisms, pixel floor, orange and teal palette, readable playable layout, no real website names, no brands, no readable text
```

视频提示词示例：

```text
中文：6秒游戏宣传片镜头，玩家冲进弹窗游乐厅，多个窗口突然弹出遮住路线，玩家击碎窗口并绕开闪光按钮陷阱，远处BOSS戴着硬币王冠升起，动作清晰，无文字

English: 6-second game trailer shot, the player rushes into a pop-up arcade hall, multiple windows burst open and block the route, the player shatters them and avoids glowing button traps, a boss wearing a coin crown rises in the distance, clear readable action, no text
```

### 3.2 放映、黑屏与缺帧记忆

| 记忆意象 | 情绪含义 | 玩法机制 | 原创命名 |
|----------|----------|----------|----------|
| 雪花屏 | 期待中断 | 局部迷雾和噪点伤害区 | 雪花雾 |
| 黑屏 | 失去观看权 | 短时间隐藏地图，需要记忆安全路径 | 静屏 |
| 缺集缺帧 | 故事不完整 | 收集胶片片段，补齐道路或剧情 | 缺帧胶卷 |
| 遥控器争夺 | 控制权被夺走 | 玩家输入短暂反转或锁定 | 遥控波 |
| 放映机光束 | 重新看见 | 光束照到的敌人显形 | 胶片灯 |
| 色彩失真 | 童年颜色被剥夺 | 切换红蓝黄通道显现机关 | 三色通道 |

设计要点：

- 黑屏不是纯惩罚，应该鼓励玩家提前观察和记忆。
- 色彩通道能做成真正的解谜和战斗机制。
- 放映厅的奖励应让玩家在其他章节也能“照出隐藏物”。

图片提示词示例：

```text
中文：原创雪花放映厅场景，巨大银幕释放黑白噪点，放映机光束照亮彩色道路，缺帧胶卷漂浮在空中，蓝紫色暗光，2D俯视角动作RPG，清晰关卡结构，无真实节目角色，无台标，无文字

English: original static projection hall scene, a giant screen releasing black-and-white noise, projector beams revealing colorful paths, missing film frames floating in the air, blue-violet low light, top-down 2D action RPG, clear level structure, no real show characters, no TV logos, no text
```

### 3.3 披风、变身与英雄幻想

| 记忆意象 | 情绪含义 | 玩法机制 | 原创命名 |
|----------|----------|----------|----------|
| 披床单当披风 | 想成为英雄 | 短时变身，提升救援和护盾能力 | 纸披风 |
| 玩具盔甲 | 童年自我保护 | 装备临时护甲，破碎后产生冲击波 | 星盾片 |
| 廉价爆炸特效 | 热血幻想被糊弄 | 假爆炸和真伤害混合，需要辨别 | 塑料爆破 |
| 英雄集结 | 不再孤单 | 救出 NPC 后加入场地支援 | 星盾队 |
| 巨大投影敌人 | 声势大但空心 | 攻击投影无效，需找真实核心 | 空壳巨像 |

设计要点：

- “强”的主题不能只做高伤害，要做救援、保护和承担。
- 假特效适合做真假判定，不要只做视觉笑点。
- 英雄基地应强调玩家从“想变强”变成“想保护”。

### 3.4 童话与被改写的结局

| 记忆意象 | 情绪含义 | 玩法机制 | 原创命名 |
|----------|----------|----------|----------|
| 魔镜 | 被评价和比较 | 映出玩家当前童心状态，生成选择 | 回声镜 |
| 诅咒苹果 | 看似甜美的坏选择 | 吃下获得增益但改变结局条款 | 甜毒果 |
| 丢失舞会 | 错过机会 | 限时路径，失败后可用书签重写一次 | 午夜页 |
| 睡眠诅咒 | 停滞 | NPC 或机关沉睡，需要找回信念碎片 | 长眠藤 |
| 结局被删 | 希望被剥夺 | 玩家收集条款重写关卡规则 | 命运书签 |

设计要点：

- 童话章节的选择不应只有“正确答案”，而应体现“还愿不愿意相信”。
- 改写结局可以直接影响地图规则，例如门、桥、天气和 NPC 状态。
- BOSS 可以临时篡改玩家技能描述，让剧情和机制合一。

图片提示词示例：

```text
中文：原创反面童话森林，水彩故事书地形，黑色墨水藤蔓覆盖道路，发光命运书签正在改写一座断桥，粉金色公主站在纸页边缘，温暖又危险，无经典童话角色直接形象，无文字

English: original reversed fairy-tale forest, watercolor storybook terrain, black ink vines covering paths, a glowing fate bookmark rewriting a broken bridge, a pink-and-gold princess standing at the edge of a paper page, warm yet dangerous, no direct classic fairy-tale character likenesses, no text
```

### 3.5 校园、课间与试卷压力

| 记忆意象 | 情绪含义 | 玩法机制 | 原创命名 |
|----------|----------|----------|----------|
| 课间十分钟 | 短暂自由 | 10 秒探索窗口，完成快速目标 | 下课铃 |
| 试卷 | 被评价 | 纸面弹幕、错题怪、红笔判定圈 | 试卷墙 |
| 排名榜 | 比较压力 | 玩家越靠近越掉童心，需绕行或破坏 | 名次柱 |
| 家长签字 | 无法逃避 | 追踪印章怪，命中后锁技能 | 签字印 |
| 春游取消 | 期待落空 | 被锁住的彩色出口，通关后恢复 | 空座巴士 |
| 暑假作业 | 拖延和愧疚 | 倒计时任务，未完成转化为追击怪 | 暑末包 |
| 小卖部 | 奖励和社交 | 补给、支线、童心恢复 | 课间柜台 |

设计要点：

- 学校章节不能只做“讨厌学习”。它的主题应是夺回学习中的好奇和课间自由。
- 错题应能变成平台或线索，表达“失败也能帮你前进”。
- 课间倒计时适合形成强节奏，是垂直切片里最容易出效果的机制之一。

```text
English: original bell schoolyard, red running track, green court, giant paper walls and red-pen judgment circles, a glowing recess bell at the center, student silhouettes running toward a colorful recess counter, top-down 2D action RPG, readable playable layout, no real school names, no text
```

### 3.6 童谣、节拍与跑调噪音

| 记忆意象 | 情绪含义 | 玩法机制 | 原创命名 |
|----------|----------|----------|----------|
| 拍手歌 | 共同节奏 | 双拍机关，需要按节奏踩格 | 拍手桥 |
| 跑调 | 熟悉感被扭曲 | 输入偏移、技能冷却错位 | 跑调波 |
| 摇铃 | 安抚 | 短时沉默敌人和噪音区 | 节拍铃 |
| 复读机 | 重复 | 录制玩家动作，影子重复一次 | 回放匣 |
| 噪声 | 心烦 | 画面震动、方向反转、童心缓慢下降 | 黑噪云 |

设计要点：

- 音乐章节可以不做复杂音游，但必须有“看得见的节拍”。
- 节奏失败不应频繁硬惩罚，可改成路线变化或敌人强化。
- 声音类提示词要描述视觉化音波，否则图像生成会虚。

### 3.7 漫画、分镜与断更

| 记忆意象 | 情绪含义 | 玩法机制 | 原创命名 |
|----------|----------|----------|----------|
| 分镜格 | 世界被切成片段 | 分镜边框成为墙或门 | 分镜墙 |
| 速度线 | 燃和冲刺 | 沿速度线快速移动 | 热血线 |
| 空白页 | 未完成 | 玩家用墨水生成桥或掩体 | 空页格 |
| 回忆篇 | 进度停滞 | 被困 NPC 重复同一句，需要补完页面 | 回忆笼 |
| 未完待续 | 等待落空 | BOSS 给自己加不可攻击护盾 | 断页幕 |

设计要点：

- 断更主题要变成“进度被冻结”的机制。
- 玩家补完最后一格时，应该真正改变地图或 BOSS 状态。
- 黑白漫画城可以用少量强调色表现童心恢复。

图片提示词示例：

```text
中文：原创断格漫画城，黑白分镜街道，速度线变成发光通路，空白漫画格被墨水画成桥，黑发公主挥动分镜笔，俯视角2D动作RPG，网点阴影，少量红色强调，无已有漫画角色

English: original broken-panel comic city, black-and-white panel streets, speed lines becoming glowing paths, blank comic panels drawn into bridges with ink, a dark-haired princess swinging a panel pen, top-down 2D action RPG, halftone shadows, small red accent color, no existing comic characters
```

### 3.8 好奇、实验与标准答案

| 记忆意象 | 情绪含义 | 玩法机制 | 原创命名 |
|----------|----------|----------|----------|
| 试管 | 好奇和尝试 | 元素混合生成不同效果 | 问号试管 |
| 实验失败 | 失败也有价值 | 错误组合生成临时平台或敌人 | 错误桥 |
| 标准答案 | 被限制 | 只有一个安全答案，但非标准答案可反击 | 标准格 |
| 放大镜 | 观察 | 显示隐藏机关和敌人弱点 | 好奇镜 |
| 电路 | 连接关系 | 连线谜题，导通后开门 | 星线圈 |

设计要点：

- 科学章节不应该惩罚试错，而应奖励观察和假设。
- “错误桥”是这个章节最重要的表达：失败不是死路。
- BOSS 的弱点应来自玩家做出的非标准实验。

### 3.9 长大压力与灰钟局

| 记忆意象 | 情绪含义 | 玩法机制 | 原创命名 |
|----------|----------|----------|----------|
| 打卡 | 被时间管理 | 经过打卡门后倒计时开始 | 灰钟门 |
| 表格 | 被格式化 | 地图变成网格，行动只能按格 | 格式地板 |
| 电梯 | 无尽上升 | 重复房间循环，需要保留童心线索破局 | 催熟电梯 |
| “快点” | 被催促 | 敌人加速，玩家童心缓慢下降 | 催促波 |
| 灰色办公室 | 想象力消退 | 饱和度降低，隐藏路线消失 | 灰域 |
| 标准成长路径 | 只有一种答案 | 多选门实际通向同一区域，需打破规则 | 单选厅 |

设计要点：

- 终章不能表达“长大是坏事”，而要表达“被格式化的长大是坏事”。
- 灰钟局的敌人应像规则、流程、表格和倒计时，而不是普通怪物。
- 真结局应允许主角长大，同时保留返回童心层的能力。

- 每日童心恢复。
- 章节回放入口。
- 公主谈心事件。
- 技能增强或隐藏路线提示。

## 5. BOSS 设计模板

每个 BOSS 至少包含三层：

```text
1. 记忆压迫：它代表什么童年压力？
2. 玩法规则：它如何改变玩家操作、视野、时间或选择？
3. 情绪反转：击败它后，玩家夺回了什么能力？
```

示例：

| BOSS | 记忆压迫 | 核心机制 | 击败后的反转 |
|------|----------|----------|--------------|
| 六元冠冕 | 快乐被诱导消费打断 | 弹窗遮挡 + 变强按钮扣童心 | 玩家学会拒绝诱导，夺回自由玩 |
| 静屏馆长 | 故事被黑屏中断 | 黑屏隐藏地图 + 光束显形 | 玩家夺回观看和记忆 |
| 补课总监 | 自由时间被占用 | 上课/课间状态切换 | 玩家夺回十分钟自由 |
| 断更书记 | 故事永远停在中途 | 回滚进度 + 空白格 | 玩家亲手补完结局 |
| 标准答案博士 | 好奇被标准化 | 单答案陷阱 + 错误桥 | 玩家证明错误也能创造道路 |
| 灰钟局长 | 成长被格式化 | 机制标准化 + 颜色归零 | 玩家保留童心地长大 |

## 6. 图片提示词规范

### 6.1 通用结构

```text

→ 角色动作
→ 机制触发
→ 场景变化
```

## 10. 自检清单

每次新增怀旧元素前，检查以下问题：

1. 它是否避开了现实作品名、角色名和品牌名？
2. 玩家不看说明，能否从场景和机制理解它在致敬什么体验？
3. 它是否提供了新的操作、规则、选择或反馈？
4. 它是否影响童心值、公主羁绊、基地回报或 BOSS 战？
5. 图片提示词是否描述了可见玩法，而不只是气氛？
6. 视频提示词是否描述了动态变化，而不只是静态镜头？
7. 这个元素是否服务当前章节闭环，而不是为了“我也知道这个梗”？

结论：怀旧不是把旧名字搬进游戏，而是把玩家曾经经历过的快乐、等待、打断、委屈和想象，做成今天还能亲手玩的规则。
