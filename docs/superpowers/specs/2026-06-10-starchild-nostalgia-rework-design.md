# 《星愿之子》怀旧主题重构设计

## 1. 范围与已确认决策

本设计用于把当前偏测试性质的内容，重构成可持续扩展的正式游戏方向。设计依据：

- `doc/怀旧主题-完整剧情.md`
- `doc/怀旧主题-主角与公主.md`
- `doc/怀旧主题-童心值动态技能.md`

已确认决策：

- 基于现有架构、框架改写，不推倒重来。
- 采用“秘密基地 Hub + 章节领域副本”结构。
- 先完整设计全游戏，再优先实现“序章 + 第一章”垂直切片。
- 角色、怪物、Boss 主体原创化；怀旧致敬只放在道具描述、闲聊台词、隐藏贴纸、海报、成就和彩蛋房。
- 视觉采用现代发光像素风。
- 第一轮使用程序化发光像素临时表现，不先建设正式 sprite 素材管线。

第一轮实现目标：

- 主菜单。
- 游戏内菜单。
- 属性面板。
- 背包系统。
- 任务日志。
- 童心值动态技能档位。
- 序章。
- 秘密基地 Hub。
- 第一章弹窗游乐厅。
- 第一章 Boss 六元冠冕。
- 覆盖上述内容的存档读档。

## 2. 游戏结构

游戏采用 Hub 循环。玩家在秘密基地休息、保存、整理背包、查看任务、和艾莉娅互动；从基地地图桌进入章节领域。每章包含探索、拾取、章节机制、小怪战、伙伴救出、Boss 战和回基地奖励。

完整章节顺序：

1. 序章：星星糖穿越
2. 第一章：弹窗游乐厅
3. 第二章：雪花放映厅
4. 第三章：纸皮英雄基地
5. 第四章：反面童话森林
6. 第五章：铃声操场
7. 第六章：跑调童谣镇
8. 第七章：断格漫画城
9. 第八章：问号实验室
10. 隐藏章：回声音乐厅
11. 终章：灰钟塔

每章循环：

1. 在基地地图桌接主线。
2. 进入章节领域。
3. 认识领域 NPC 或被困伙伴。
4. 收集三个或更多章节关键物。
5. 学会本章机制。
6. 救出或唤醒伙伴。
7. 触发 Boss。
8. 击败 Boss。
9. 获得领域信物。
10. 回基地安放信物。
11. 解锁新基地功能、入口或章节门。

## 3. 第一轮垂直切片

第一轮垂直切片要证明正式循环能跑通：

1. 从主菜单开始新游戏。
2. 完成序章并遇见艾莉娅。
3. 进入秘密基地。
4. 打开任务、属性、背包、系统菜单页。
5. 从基地地图桌接第一章。
6. 进入弹窗游乐厅。
7. 收集试玩币和材料。
8. 救出铁翼。
9. 击败六元冠冕。
10. 带像素手柄回基地。
11. 看到基地和伙伴状态变化。
12. 保存并读回同一进度。

第一轮实际实现区域：

- `real_street_prologue`
- `home_base`
- `popup_arcade`

其他章节先作为设计内容保留，不进入第一轮可玩范围。

Boss 触发规则：

1. 章节关键物收集完成。
2. 章节伙伴被救出或觉醒。
3. 章节核心机关被启动。

Boss 设计规则：

- 至少一个本章地图机制。
- 至少一个诱导或压力机制。
- 至少一个清晰反制点。
- 至少一个高童心奖励条件。

### 第一章机制

弹窗遮挡：

- 怪物命中或机关触发后，屏幕局部出现像素弹窗。
- 玩家攻击场景里的关闭 X 机关可以清理弹窗。
- 不处理弹窗时，弹窗会在短时间后自然消失。

诱导按钮：

- 站上去短时间攻击 +20%。
- 立即扣除 40 童心值。
- 隐藏贴纸要求少踩或不踩诱导按钮。

试玩币机关：

- 收集三枚试玩币后 Boss 门亮起。
- 第一章通关后，该机关出现在基地，成为训练房入口。

铁翼救出：

- 第一轮铁翼不做完整常驻战斗 AI。
- Boss 开场时，铁翼用剧情支援打碎第一层无敌护盾。

### 六元冠冕 Boss

Boss 形象：

- 漂浮皇冠形广告牌。
- 脸由像素金币、倒计时和关闭 X 按钮组成。
- 重点不是高伤害，而是诱导、屏幕压力和安全区判断。

触发条件：

1. 收集三枚试玩币。
2. 击败废铁工厂精英怪并救出铁翼。
3. 用铁翼火箭拳打碎弹窗护盾核心。

阶段：

| 阶段 | 血量 | 机制 | 攻击 |
|---|---:|---|---|
| 欢迎弹窗 | 100%-70% | 屏幕边缘弹窗遮挡 | 横向优惠券弹幕、慢速弹窗泡泡 |
| 限时优惠 | 70%-35% | 诱导按钮出现 | 地面红框、金币雨、关闭叉冲刺 |
| 小窗口世界 | 35%-0% | 场地分成 4 个窗口，只有亮起窗口安全 | 窗口切换、全屏灰雾、皇冠激光 |

反制方式：

- 攻击 Boss 本体造成普通伤害。
- 攻击关闭 X 机关可清理遮挡并让 Boss 短暂硬直。
- 铁翼打破第一次无敌护盾。
- 连续使用诱导按钮会强化 Boss 并降低童心。
- 童心值高于 500 通关可获得隐藏贴纸。

奖励：

- 像素手柄。
- 铁翼入队。
- 童心 +120。
- 金币 +60。
- 像素螺丝、彩色电池、旧按键。
- 可选隐藏奖励：`未充值也能赢贴纸`。

检查点：

| 检查点 | 触发 |
|---|---|
| CP1 | 进入弹窗游乐厅 |
| CP2 | 收集第 2 枚试玩币 |
| CP3 | 救出铁翼 |
| CP4 | Boss 战前 |
| CP5 | Boss 击败 |

## 10. 童心值动态技能与战斗

第一轮要让玩家通过以下反馈直接感知童心值：

- 技能名称。
- 伤害倍率。
- 粒子强度。
- UI 边框颜色。
- 画面饱和度。
- 隐藏物可见性。

### 档位表

| 档位 | 数值 | 世界表现 | 技能表现 |
|---|---:|---|---|
| 失色 | 0-199 | 画面泛灰，移动变慢 | 伤害低、粒子少、名称朴素 |
| 寻常 | 200-499 | 正常色彩 | 标准技能 |
| 鲜活 | 500-799 | 色彩更亮，隐藏物可见 | 伤害提高、粒子更亮 |
| 绚烂 | 800-1000 | 边缘发光，彩色粒子 | 最高伤害，范围略增 |

### 第一轮技能

沿用现有技能概念，并接入童心档位。

| 输入 | 技能族 | 失色 | 寻常 | 鲜活 | 绚烂 |
|---|---|---|---|---|---|
| 火系输入 | 火系投射 | 打火机 | 火球术 | 流星坠 | 星愿焰火 |
| 冰系输入 | 冰系投射 | 冷风 | 冰锥刺 | 霜花爆裂 | 童心冻结 |
| 雷系输入 | 连锁闪电 | 静电 | 闪电链 | 雷暴交响 | 童心共鸣雷 |
| `F` | 防御 | 抱头 | 护盾 | 星辉屏障 | 童心圣域 |
| `Space` | 位移 | 快步 | 飞行 | 星翼滑翔 | 星愿翱翔 |

第一轮机制：

- 火系：投射物伤害、速度、颜色、小范围爆炸半径随档位变化。
- 冰系：命中减速，高档位减速更强。
- 雷系：连锁目标数量随档位变化。
- 防御：减伤比例随档位变化。
- 位移：距离、冷却、跨越低矮障碍能力随档位变化。
- 被动：击败敌人恢复少量童心；基地恢复更强。

不进入第一轮的高级机制：

- 火球分裂。
- 火焰领域。
- 冰面滑行。
- 雷域。
- 完整友方光环。
- 完整伙伴羁绊技。

战斗规则：

- 生命归零：回到秘密基地床边，增加委屈，恢复到最近检查点。
- 童心归零：触发失色保护流程，强制回基地，并设置坏结局风险 flag。
- 委屈过高：移动变慢，童心自然恢复变慢。
- 高童心击败 Boss：可给贴纸或童心上限。
- 诱导机制：提供短期战斗收益，同时造成长期童心损失。

## 12. 架构设计

本次重构扩展现有系统边界。

### 保留并扩展的系统

| 现有系统 | 继续承担 | 扩展方向 |
|---|---|---|
| `RegionManager / RegionFactory` | 区域加载和切换 | 正式区域 ID、章节门、Boss 房入口 |
| `QuestSystem` | 任务状态与奖励 | 目标列表任务 |
| `EmotionSystem` | 童心与委屈 | 档位 helper、技能倍率、隐藏物可见 |
| `Inventory` | 金币和家具库存 | 通用物品堆叠与领域信物 |
| `SaveSystem` | 序列化和迁移 | 剧情进度、背包、伙伴、Boss 状态 |
| `Presentation` | 渲染分层 | 菜单、面板、HUD、像素临时表现 |
| `EnemyManager` | 普通怪生命周期和 AI | 怪物定义和特殊命中效果 |
| `DialogueService` | 对话脚本和奖励 | 主线 NPC、艾莉娅、伙伴对话 |

### 新增边界

| 新单元 | 职责 |
|---|---|
| `StoryProgress` | 章节状态、伙伴解锁、结局 flag、隐藏章 flag |
| `ItemCatalog` | 物品 ID、名称、分类、描述、可用性、使用效果 |
| `ChapterDefinition` | 章节区域、进入条件、Boss、关键物、通关奖励 |
| `BossController` | Boss 血量、阶段、冷却、胜负回调 |
| `PopupCrownBoss` | 第一章 Boss 专属行为 |
| `PixelActorView` | 程序化像素角色、NPC、怪物和 Boss 表现 |
| `GameMenuView` | 游戏内菜单框架 |
| `QuestLogView` | 任务页 |
| `CharacterPanelView` | 属性页 |
| `InventoryView` | 背包页 |
| `PartnerPanelView` | 伙伴页 |

### 数据流

启动流程：

1. `MainMenuScene` 开新游戏或读档。
2. `WorldScene` 加载当前状态。
3. `RegionManager` 加载区域。
4. `StoryProgress` 判断章节可用性。
5. `QuestSystem` 更新任务状态。
6. `Inventory` 提供物品状态。
7. `PresentationModelBuilder` 组装 HUD 和菜单模型。
8. `SaveSnapshotBuilder` 保存完整状态。

战斗流程：

1. 输入进入技能控制器。
2. `EmotionSystem` 提供当前童心档位。
3. 技能和战斗服务应用档位数值。
4. 普通怪通过 `EnemyManager` 更新。
5. Boss 通过 `BossController` 更新。
6. 掉落由 `DropManager` 生成。
7. 拾取回调写入 `Inventory`。
8. 任务目标由 `QuestSystem` 更新。

### 配置策略

第一轮不过度泛化：

- 物品定义可先用 C++ 静态表或轻量 Lua 表。
- 第一轮任务使用扩展后的 `QuestSystem`。
- 第一章 Boss 使用 C++ 专属类，不提前做 Boss 脚本引擎。
- 章节区域先在 `RegionFactory` 显式创建。
- UI 第一轮保持程序化绘制。

## 14. 第一轮非目标

第一轮不包含：

- 第二章到第八章完整可玩内容。
- 隐藏章实装。
- 终章实装。
- 正式 sprite 素材管线。
- 完整伙伴常驻战斗 AI。
- 完整装备系统。
- 完整商店经济。
- 完整设置菜单。
- 完整 Boss 脚本编辑器。

## 15. 第一轮完成定义

第一轮完成时，玩家可以：

1. 开始新游戏。
2. 完成序章并遇见艾莉娅。
3. 进入秘密基地。
4. 打开任务、属性、背包、系统菜单。
5. 接第一章任务。
6. 探索弹窗游乐厅。
7. 收集试玩币和材料。
8. 救出铁翼。
9. 击败六元冠冕。
10. 带像素手柄回基地。
11. 看到基地和伙伴状态变化。
12. 保存、退出、继续游戏，并恢复同一进度。
